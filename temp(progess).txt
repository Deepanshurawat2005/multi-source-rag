==================================================
PDF LOADING
===========

Load ALL PDFs from:

data/uploaded_pdfs/

Not a single fixed PDF.

Create:

load_all_pdfs(folder_path)

Logic:

* Scan entire folder
* Find all .pdf files
* Load each PDF using PyPDFLoader
* Merge all pages into one documents list
* Return combined documents

Example:

for every PDF:
loader = PyPDFLoader(pdf_path)
docs.extend(loader.load())

==================================================
IMPORTANT PDF LOGIC
===================

DO NOT blindly trust PDF answer.

Before returning PDF answer:

Check if answer contains phrases like:

* there is no mention
* not mentioned
* not present
* cannot be found
* unable to provide
* not available in the context
* not found in the context

If such phrases exist:

Treat PDF answer as failure.

Then automatically route to:

Wikipedia / Arxiv / Web

instead of returning PDF answer.

Example:

Question:
Who is Mahatma Gandhi?

PDF:
"There is no information about Mahatma Gandhi in the provided context."

Result:

DO NOT return PDF answer.

Instead:

Go to Wikipedia.

==================================================
SOURCE DISPLAY RULE
===================

Frontend already displays source badge.

Therefore:

Do NOT write:

"According to the provided context"

Do NOT write:

"Based on the provided context"

Do NOT write:

"Unfortunately the context is insufficient"

Do NOT write:

"The context mentions"

Return direct final answer only.

Example:

Bad:

According to the provided context, Mahatma Gandhi was...

Good:

Mahatma Gandhi was the leader of India's freedom movement against British rule...

==================================================
PDF UPLOAD BEHAVIOR
===================

When new PDF uploaded:

1. Save PDF
2. Load all PDFs
3. Create chunks
4. Rebuild FAISS
5. Rebuild retriever
6. Rebuild rag chain

This ensures newly uploaded PDFs become searchable immediately.

==================================================
PDF DELETE BEHAVIOR
===================

When PDF deleted:

1. Delete file from folder
2. Reload all PDFs
3. Recreate chunks
4. Recreate FAISS
5. Recreate retriever
6. Recreate rag chain

This ensures deleted PDFs disappear from retrieval.

==================================================
PAGE REFRESH BEHAVIOR
=====================

When frontend loads:

Call:

GET /api/pdfs

Populate state.pdfs

Render PDF cards.

Result:

Uploaded PDFs remain visible even after page refresh.

==================================================
ERROR HANDLING
==============

If no PDFs uploaded:

Do NOT stop chat.

Skip PDF retrieval.

Directly route to:

Wikipedia / Arxiv / Web.

Chat must always work.
